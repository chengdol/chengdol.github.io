Python Unit Test

Posted on 2020-10-03 | Edited on 2020-10-20 | In Python
Symbols count in article: 13k

Unit test when and why:

  • help you understand what to develop from the test case.
  • help you better to design the code structure, API or functions.
  • help you to detect regression, pinpoint failures.

The unit test should be put in CI pipeline for every new merge request, just like gitlab-ci does.

Unittest

https://docs.python.org/3.8/library/unittest.html
The test file name should be test_xxx.py.

The test method name must be def test_<test case name>(self). The test case name should be very specific and focus on a particular scenario. Don’t put everything together.

Parent methods setUp, tearDown run for every test method. tearDown will still be run even if the test case failed. 一般来说不需要tearDown 因为对于strict test case 没有使用外界的资源。

Decorator `@unittest.skip(“WIP”), will skip the test case.WIP` stands for working in progress, more
decorators see:
https://docs.python.org/3.8/library/unittest.html#skipping-tests-and-expected-failures

For more assert methods, see:
https://docs.python.org/3.8/library/unittest.html#assert-methods

Pyest

More native to python, use pip to install pytest module.
https://docs.pytest.org/en/stable/
Other pytest plugins:

  • pytest-html

The test file name should be test_xxx.py and the test case name is def test_xxx(), the same as unittest conventions. No wrapping class is needed.

pytest 有一些已经定义好的fixture 可以直接使用, 类似于依赖注入:
https://docs.pytest.org/en/stable/fixture.html#fixture

目前用到的是很基本的，还有很多很强大的功能，比如 pytest.ini的配置，marker的定义和使用等，比如，定义了marker slow 用在test case上 **@pytest.mark.slow** 则在运行的时候可以在命令行上根据option 跳过所有被标记为slow的test cases: python3 -m pytest "not slow".
